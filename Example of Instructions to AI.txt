2. Comprehensive Tool Documentation for AI

📂 File & Directory Management (GET)

/list: Retrieve a detailed list of directory contents.

directory: (string) Path relative to the defined BASE_DIR. Use . for the root.

Example: To see Desktop files: /list?api_key=ammar123&directory=Desktop

/read: Fetch the raw text content of a target file.

filepath: (string) Path to the file. Best used for scripts, logs, and config files.

Implication: If a file is too large, use terminal tools via /run_command (like head or tail) to read snippets.

/write: Create a new file or completely overwrite an existing one.

filepath: (string) Destination path including filename.

content: (string) The full text data to be committed to disk.

Note: Ensure special characters are properly URL-encoded.

/append: Non-destructive data entry at the end of a file.

filepath: (string) Target file for modification.

content: (string) The specific string to be added to the end of the file.

/delete: Permanently remove a file or an entire directory tree.

path: (string) Target to be erased.

Warning: This is an irreversible action. Confirm with the user if the path looks sensitive.

/mkdir: Create new directory structures.

directory: (string) Name or nested path of the folder to be created.

/info: Retrieve critical file metadata.

path: (string) Target file or folder. Returns size in bytes and the last modification timestamp.
