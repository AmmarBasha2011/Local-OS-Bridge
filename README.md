# 🚀 Ultimate Local OS Bridge (V5 RAT)

### النظام المتكامل لإدارة وتحكم أنظمة Linux عبر الذكاء الاصطناعي والـ API المتقدم

---

## 📝 وصف المشروع وتصوره العام

يعد مشروع **Ultimate Local OS Bridge** بمثابة "دماغ رقمي" وسيط، صُمم خصيصاً لتمكين نماذج الذكاء الاصطناعي المتطورة (مثل Gemini, GPT-4, Claude) من التفاعل المباشر والعميق مع بيئة نظام تشغيل Linux (تحديداً **Ubuntu MATE**). بدلاً من الاكتفاء بالدردشة النصية، يتحول الذكاء الاصطناعي إلى "مدير نظام" حقيقي قادر على تنفيذ المهام، تحليل البيانات، وإدارة الشبكة بشكل حيوي.

تم بناء هذا الـ API باستخدام إطار عمل **FastAPI** لضمان سرعة استجابة فائقة (Low Latency) ومعالجة متزامنة للطلبات، مما يجعله أداة قوية جداً للمطورين وخبراء الأمن السيبراني الراغبين في أتمتة عملياتهم المنزلية.

> **⚠️ إخلاء مسؤولية وأهداف المشروع:** تم تطوير هذا المشروع بواسطة المطور **عمار الخطيب** كأداة للبحث العلمي، التعلم العميق، والاختبار الأمني الأخلاقي داخل بيئة منزلية مغلقة. يُحظر تماماً استخدامه في أي نشاط غير قانوني أو خارج الشبكات المصرح لك بالوصول إليها.

## 🛠️ الهندسة التقنية: كيف يعمل الكود؟

يعتمد النظام على هيكلية هجينة تجمع بين محرك الويب المحلي والوصول السحابي الآمن:

1.  **المحرك الأساسي (Backend):** يعمل الخادم عبر `Uvicorn` على المنفذ `8000` محلياً، حيث يقوم بمعالجة الطلبات الواردة وتحويلها إلى أوامر نظام (System Calls) باستخدام مكتبات Python الأساسية مثل `os` و `subprocess`.
2.  **نظام الأمان المزدوج:**
    * **مفتاح الوصول (API Key):** حارس البوابة الأول، حيث يتم التحقق من `API_KEY` (القيمة: `ammar123`) في كل طلب.
    * **تتبع العمليات (Middleware):** يقوم النظام بتسجيل كل حركة (Method, Path, Status) في الوقت الفعلي على شاشة الـ Terminal لمراقبة سلوك الذكاء الاصطناعي.
3.  **الاتصال الخارجي (Tunneling):** عبر مكتبة `pycloudflared` يتم إنشاء نفق مشفر يتخطى جدران الحماية المحلية (Firewalls) ويوفر رابطاً مؤقتاً (HTTPS) يتيح للذكاء الاصطناعي البعيد الوصول للجهاز دون الحاجة لفتح منافذ في الراوتر (Port Forwarding).
4.  **حماية المسارات (Strict Sandboxing):** تستخدم دالة `get_safe_path` تقنية `os.path.realpath` للتأكد من أن أي عملية ملفات لا تخرج عن نطاق المجلد `BASE_DIR`. هذا يمنع أي محاولة من الذكاء الاصطناعي للوصول لملفات النظام الحساسة مثل `/etc/shadow`.

## 🚀 الدليل الشامل لنقاط النهاية (50+ Endpoints)

### 📂 1. العمليات الجوهرية للملفات
* `GET /list`: سرد محتويات المجلدات مع دعم المسارات النسبية.
* `GET /read`: استخراج النصوص من الملفات مع معالجة الأخطاء والترميز.
* `GET /write`: إنشاء ملفات جديدة أو استبدال المحتوى بالكامل.
* `GET /append`: إضافة سجلات أو بيانات لنهاية الملفات دون مسح القديم.
* `GET /delete`: تدمير الملفات أو المجلدات (استخدمه بحذر!).
* `GET /mkdir`: بناء هيكلية مجلدات معقدة.
* `GET /copy` & `/rename`: نقل وتكرار البيانات داخل النظام.

### 🔬 2. الإدارة المتقدمة والتحليل الفني
* `GET /info`: تحليل الميتا-داتا (الحجم، تاريخ التعديل، الأذونات).
* `GET /find_ext`: محرك بحث ذكي لإيجاد أنواع ملفات محددة في المجلدات الفرعية.
* `GET /split` & `/merge`: ميزة حيوية للتعامل مع الملفات الضخمة التي لا تستطيع نافذة الذكاء الاصطناعي استيعابها دفعة واحدة.
* `GET /symlink`: إنشاء اختصارات برمجية لربط المجلدات.

### 📊 3. معالجة البيانات والبحث الذكي
* `GET /grep`: محاكاة لأمر Linux الشهير للبحث عن نصوص داخل آلاف الملفات.
* `GET /regex_search`: استخدام التعبيرات النمطية (Regex) لاستخراج الإيميلات، الأرقام، أو الأنماط البرمجية.
* `GET /diff`: أداة مقارنة برمجية لمعرفة التغييرات بين نسختين من ملف واحد.
* `GET /csv_to_json` & `/json_to_csv`: جسر بيانات لتحويل الجداول إلى كائنات برمجية والعكس لسهولة التحليل.

### 💻 4. التحكم الكامل في النظام (OS Power)
* `GET /run_command`: الصلاحية الكاملة لتنفيذ أوامر Shell (مثل تثبيت برامج، تحديث النظام).
* `GET /kill`: أداة الطوارئ لإنهاء العمليات المعلقة عبر PID.
* `GET /ps`: عرض حي لأداء النظام (أكثر العمليات استهلاكاً للـ CPU والرام).
* `GET /git_status`: أداة للمطورين لمتابعة حالة المشاريع البرمجية مباشرة.

### 🌐 5. ترسانة الشبكة والاتصال
* `GET /download_url`: تحويل الجهاز إلى خادم تحميل (Seed box).
* `GET /curl`: إرسال طلبات ويب متقدمة، مما يسمح للذكاء الاصطناعي باستخدام جهازك كـ Proxy.
* `GET /ip`: التحقق من الهوية الرقمية للجهاز على الإنترنت.
* `GET /ports`: كاشف المنافذ للتأكد من الخدمات التي تعمل في الخلفية.

### 🔐 6. التشفير والأرشفة
* `GET /hash` & `/md5`: التأكد من سلامة الملفات وعدم تعرضها للتلاعب (Integrity Check).
* `GET /base64_read/write`: الطريقة المثلى لنقل ملفات الصور والملفات الثنائية عبر الـ API.
* `GET /tar` & `/gzip`: أدوات الضغط لتقليل حجم البيانات قبل نقلها أو تخزينها.

## ⚙️ متطلبات التشغيل والتثبيت

لضمان عمل النظام بكفاءة على **Ubuntu MATE**، اتبع الخطوات التالية:

1.  **تجهيز البيئة:**
    ```bash
    sudo apt update && sudo apt install python3-pip -y
    pip install fastapi uvicorn pycloudflared requests
    ```
2.  **التشغيل:**
    ```bash
    python3 main.py
    ```
3.  **الربط:** بمجرد التشغيل، سيقوم الكود بطباعة رابط `*.trycloudflare.com`. قم بتزويد هذا الرابط لمساعدك الذكي مع الـ API Key لبدء التحكم.
